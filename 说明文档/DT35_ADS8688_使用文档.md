# DT35 激光测距传感器 + ADS8688 ADC 驱动使用文档

## 1. 系统概述

```
DT35 激光传感器              ADS8688 (16-bit ADC)             STM32H723
  ┌──────────┐              ┌──────────────────┐            ┌──────────────┐
  │          │  0~10V 模拟  │                  │   SPI3     │              │
  │  DT35    ├─────────────>│  CH0             │            │              │
  │          │              │                  │            │              │
  │  50mm    │              │         SCK  ────┼── PC10 ───>│  SPI3_SCK    │
  │  ~       │              │         SDI  ────┼── PC12 ───>│  SPI3_MOSI   │
  │  10000mm │              │         SDO  ────┼── PC11 <───│  SPI3_MISO   │
  │          │              │         CS   ────┼── PE14 <───│  GPIO OUT    │
  └──────────┘              │         RST  ────┼── PB8  <───│  GPIO OUT    │
                            │                  │            │              │
                            │  Vref = 4.096V   │            │  HAL + RTOS  │
                            └──────────────────┘            └──────────────┘
```

### 信号链路

| 阶段 | 说明 |
|------|------|
| DT35 输出 | 根据被测距离输出 0~10V 模拟电压 |
| ADS8688 采集 | 通过 CH0 采集该电压, 量程设为 0~10.24V (单极性) |
| SPI3 传输 | STM32H7 通过硬件 SPI3 读取 16 位 ADC 原始值 |
| 软件换算 | ADC 原始值 → 电压 → 距离 (mm) |

## 2. 硬件接线

### 2.1 SPI3 引脚 (AF6)

| ADS8688 引脚 | STM32H7 引脚 | 功能 | 备注 |
|:------------|:------------|:-----|:-----|
| SCLK | PC10 | SPI3_SCK | 时钟, AF6 |
| SDI (DIN) | PC12 | SPI3_MOSI | 主机发送, AF6 |
| SDO (DOUT) | PC11 | SPI3_MISO | 主机接收, AF6 |

### 2.2 控制引脚 (GPIO)

| ADS8688 引脚 | STM32H7 引脚 | 功能 | CubeMX 配置 |
|:------------|:------------|:-----|:-----|
| CS (片选) | PE14 | GPIO 推挽输出 | gpio.c 已配置 |
| RST (复位) | PB8 | GPIO 推挽输出 | gpio.c 已配置 |
| DAISY | 接 GND | 菊花链禁用 | 单芯片使用接低 |

### 2.3 DT35 传感器

| DT35 引脚 | 连接目标 | 说明 |
|:----------|:---------|:-----|
| 模拟输出 (+) | ADS8688 CH0 输入 | 0~10V 模拟电压 |
| GND | ADS8688 AGND | 共地 |

## 3. CubeMX 配置 (已完成)

### 3.1 SPI3 配置 (spi.c)

CubeMX 已生成 `MX_SPI3_Init()`, 关键参数:

| 参数 | 值 | 说明 |
|:-----|:---|:-----|
| Instance | SPI3 | - |
| Mode | SPI_MODE_MASTER | 主机模式 |
| Direction | SPI_DIRECTION_2LINES | 全双工 |
| DataSize | SPI_DATASIZE_8BIT | 8 位数据 |
| CLKPolarity | SPI_POLARITY_LOW | 空闲低电平 (CPOL=0) |
| CLKPhase | SPI_PHASE_1EDGE | 第一个边沿采样 (CPHA=0) |
| NSS | SPI_NSS_SOFT | 软件片选 |
| BaudRatePrescaler | /64 | SPI 时钟分频 |
| FirstBit | SPI_FIRSTBIT_MSB | 高位先发 |

> SPI3 时钟源: PLL1, GPIO: PC10(SCK) PC11(MISO) PC12(MOSI) AF6

### 3.2 GPIO 配置 (gpio.c)

| 引脚 | 模式 | 初始电平 | 用途 |
|:-----|:-----|:---------|:-----|
| PE14 | GPIO_OUTPUT_PP, 上拉 | LOW | ADS8688 CS |
| PB8 | GPIO_OUTPUT_PP | LOW | ADS8688 RST |

### 3.3 main.c 初始化顺序

```c
MX_GPIO_Init();     // ← PE14 (CS), PB8 (RST)
MX_SPI3_Init();     // ← SPI3 硬件初始化
```

> 以上由 CubeMX 自动生成, **不需要手动修改**.

## 5. API 说明

### 5.1 头文件

```cpp
#include "DT35.h"
```

全局实例已在 `DT35.cpp` 中定义:

```cpp
extern DT35 dt35;   // 直接使用, 无需手动创建
```

### 5.2 数据结构

```cpp
struct DT35_Data_t {
    float    distance_mm;   // 距离, 单位 mm (50 ~ 10000)
    float    voltage_V;     // 电压, 单位 V  (0 ~ 10.24)
    uint16_t adc_raw;       // ADC 原始值    (0 ~ 65535)
    uint8_t  valid;         // 数据有效标志  (0=无效, 1=有效)
};
```

### 5.3 公共方法

| 方法 | 原型 | 说明 |
|:-----|:-----|:-----|
| **init** | `void init(SPI_HandleTypeDef *hspi)` | 传入 CubeMX 生成的 SPI 句柄, 初始化 ADS8688 |
| **update** | `void update(void)` | 读取一次 ADC, 更新 `data` 中的电压和距离 |

## 6. 使用示例

### 6.1 基本用法 (在 FreeRTOS 任务中)

```cpp
#include "spi.h"     // CubeMX 生成, 包含 extern hspi3
#include "DT35.h"

void lift_task(void *argument)
{
    dt35.init(&hspi3);   // 传入 SPI3 句柄

    for (;;)
    {
        dt35.update();

        if (dt35.data.valid)
        {
            float dist = dt35.data.distance_mm;   // 距离, mm
            float volt = dt35.data.voltage_V;     // 电压, V
            uint16_t raw = dt35.data.adc_raw;     // ADC 原始值

            // --- 在这里使用距离值 ---
        }

        osDelay(10);   // 10ms = 100 Hz
    }
}
```

### 6.2 配合串口打印调试

```cpp
#include "spi.h"
#include "DT35.h"
#include <stdio.h>

void lift_task(void *argument)
{
    dt35.init(&hspi3);

    for (;;)
    {
        dt35.update();

        printf("ADC=%5u  V=%.3fV  D=%.1fmm\r\n",
               dt35.data.adc_raw,
               dt35.data.voltage_V,
               dt35.data.distance_mm);

        osDelay(100);
    }
}
```

### 6.3 带阈值判断的应用

```cpp
#include "spi.h"
#include "DT35.h"

#define TARGET_HEIGHT_MM    500.0f
#define TOLERANCE_MM        5.0f

void lift_task(void *argument)
{
    dt35.init(&hspi3);

    for (;;)
    {
        dt35.update();

        if (dt35.data.valid)
        {
            float error = dt35.data.distance_mm - TARGET_HEIGHT_MM;

            if (error > TOLERANCE_MM) {
                // 距离太远, 需要下降
            } else if (error < -TOLERANCE_MM) {
                // 距离太近, 需要上升
            } else {
                // 到达目标位置
            }
        }

        osDelay(10);
    }
}
```

## 7. 工作原理详解

### 7.1 初始化流程 (`init`)

```
dt35.init(&hspi3)
    │
    ├── 1. 保存 SPI 句柄指针 (hspi3 由 CubeMX 配置)
    ├── 2. 清零数据结构
    │
    └── 3. ADS8688_Init()
            ├── CS 拉高 (空闲)
            ├── 硬件复位: RST(PB8) 拉低 1ms → 拉高
            ├── 软件复位: 发送 RST 命令, 等待 25ms
            ├── 写寄存器: 使能 CH0 (AUTO_SEQ_EN = 0x01)
            ├── 写寄存器: 关闭 CH1~CH7 电源 (CH_PD = 0xFE)
            ├── 写寄存器: CH0 量程 = 0~10.24V
            ├── 回读验证: 确认 CH0 使能成功 (最多重试 10 次)
            ├── 发送 MAN_CH0 命令: 进入手动 CH0 模式
            └── 丢弃第一次读数 (首次转换不可靠)
```

### 7.2 数据采集流程 (`update`)

```
dt35.update()
    │
    ├── 1. ADS8688_ReadADC()
    │       ├── CS 拉低
    │       ├── SPI 全双工收发 4 字节 NO_OP
    │       │     SDI: [0x00][0x00][0x00][0x00]
    │       │     SDO: [ xx ][ xx ][ADC_H][ADC_L]
    │       ├── CS 拉高
    │       └── 返回 (ADC_H << 8) | ADC_L
    │
    ├── 2. ADC → 电压
    │       voltage_V = raw * 10.24 / 65535
    │
    └── 3. 电压 → 距离
            distance_mm = 50 + V * 995
```

### 7.3 ADS8688 SPI 帧格式

每次 SPI 通信为 **32 位 (4 字节)** 全双工帧:

```
        Byte 0      Byte 1      Byte 2      Byte 3
       ┌──────────┬──────────┬──────────┬──────────┐
 SDI:  │ CMD_HIGH │ CMD_LOW  │  0x00    │  0x00    │  ← 主机发送 (命令)
       ├──────────┼──────────┼──────────┼──────────┤
 SDO:  │ (不关心)  │ (不关心)  │ ADC_HIGH │ ADC_LOW  │  ← 从机返回 (数据)
       └──────────┴──────────┴──────────┴──────────┘
```

### 7.4 寄存器操作帧格式

**写寄存器 (16 位)**:
```
 SDI:  [addr<<1 | 1] [data]
```

**读寄存器 (24 位)**:
```
 SDI:  [addr<<1 | 0] [0x00] [0x00]
 SDO:  [  不关心   ] [不关心] [data]
```

## 8. 电压-距离换算公式

### 8.1 ADC 原始值 → 电压

ADS8688 在 0~10.24V 单极性量程下:

```
电压 (V) = ADC原始值 × 10.24 / 65535
```

| ADC 原始值 | 电压 |
|:----------|:-----|
| 0 | 0 V |
| 32000 | ~5.0 V |
| 63897 | ~10.0 V |
| 65535 | 10.24 V |

### 8.2 电压 → 距离

DT35 线性映射: 0V = 50mm, 10V = 10000mm

```
距离 (mm) = 50 + 电压 (V) × 995
```

| 电压 | 距离 |
|:-----|:-----|
| 0 V | 50 mm |
| 1 V | 1045 mm |
| 5 V | 5025 mm |
| 10 V | 10000 mm |

### 8.3 ADC 精度

```
ADC 分辨率 = 10.24V / 65536 ≈ 0.156 mV/LSB
距离分辨率 = 0.156mV × 995 / 1000 ≈ 0.155 mm/LSB
```

## 9. 常见问题

### Q1: `init()` 应该在哪里调用?

在 FreeRTOS 任务函数内部, `MX_SPI3_Init()` 之后调用.

```cpp
void lift_task(void *argument) {
    dt35.init(&hspi3);    // RTOS 已启动, SPI3 已初始化
    for(;;) { ... }
}
```

### Q2: 读数全为 0 或 65535?

1. **接线**: 确认 SCK/MOSI/MISO/CS/RST 五根线连接正确
2. **供电**: ADS8688 是否正常上电
3. **DT35 信号**: 万用表测量 DT35 输出是否在 0~10V 范围
4. **SPI 波形**: 逻辑分析仪抓取 SPI 信号确认时序

### Q3: 读数跳动很大?

增加软件滤波, 多次采样取平均:

```cpp
float avg = 0;
for (int i = 0; i < 10; i++) {
    dt35.update();
    avg += dt35.data.distance_mm;
    osDelay(1);
}
avg /= 10.0f;
```

### Q4: 如何更换到其他通道?

修改 `DT35.cpp` 中 `ADS8688_Init()`:

```cpp
// 例如改为 CH3:
ADS8688_WriteReg(ADS8688_REG_AUTO_SEQ_EN, 0x08);  // bit3 = CH3
ADS8688_WriteReg(ADS8688_REG_CH_PD, ~0x08);
ADS8688_WriteReg(ADS8688_REG_CHIR_3, ADS8688_RANGE_0_10V24);
ADS8688_WriteCmd(ADS8688_CMD_MAN_CH3);
```

### Q5: 如何修改距离映射范围?

修改 `DT35.h` 中的宏:

```cpp
#define DT35_VOLTAGE_MAX_V      10.0f
#define DT35_DIST_AT_0V_MM      50.0f      // 0V 对应距离
#define DT35_DIST_AT_10V_MM     10000.0f   // 10V 对应距离
```
